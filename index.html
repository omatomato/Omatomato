Verificação de segurança Security verification

Como verificar tudo (Autenticidade + Integridade)

O autor publica um manifesto assinado que contém os hashes SHA-256 dos relatórios. Sempre confirme a impressão digital (fingerprint) da chave PGP por um canal independente antes de confiar na assinatura.

Fluxo recomendado (sem expor caminhos internos)

1. Baixe a chave pública do autor (link na seção de chave PGP).
2. Baixe o manifesto assinado fornecido pelo autor e a sua assinatura (os arquivos estarão disponíveis para download no site).
3. Verifique a assinatura do manifesto usando a chave pública do autor.
4. Se a assinatura for válida, verifique os hashes SHA-256 localmente comparando-os com os arquivos baixados.

Comandos — use nomes locais dos arquivos que você baixou

# importar chave pública (apontando para o arquivo que você baixou)
gpg --import arquivo_chave_publica.asc

# verificar assinatura do manifesto (use os nomes que você salvou)
gpg --verify manifesto_assinatura.asc manifesto.txt

# verificar integridade dos relatórios usando o manifesto local
sha256sum -c manifesto.txt

Se o GPG responder Good signature e o sha256sum devolver OK para cada item, os arquivos são autênticos em relação ao manifesto assinado.

Boas práticas

  • Confirme o fingerprint em um canal independente (Telegram, perfil público, contato verificado).
  • Faça as verificações em uma máquina limpa, preferencialmente offline.
  • Guarde manifestações assinadas com nomes datados (ex.: manifesto-2026-02-28.txt) para histórico — sem expor caminhos internos.

How to verify everything (Authenticity + Integrity)

The author publishes a signed manifest that lists SHA-256 hashes for the reports. Always confirm the PGP fingerprint via an independent channel before trusting the signature.

Recommended workflow (without exposing internal paths)

1. Download the author's public key (link in the PGP section).
2. Download the signed manifest and its signature from the author (available for download on the site).
3. Verify the manifest signature with the public key.
4. If the signature is valid, verify SHA-256 hashes locally against the downloaded reports.

Commands — use the local filenames you downloaded

# import public key (point to the file you downloaded)
gpg --import downloaded_public_key.asc

# verify manifest signature (use your downloaded filenames)
gpg --verify downloaded_manifest_signature.asc downloaded_manifest.txt

# check file integrity using the manifest
sha256sum -c downloaded_manifest.txt

If GPG returns Good signature and sha256sum returns OK for each entry, the files match the signed manifest and are authentic to the key holder.

Best practices

  • Confirm the fingerprint over an independent channel (Telegram, verified profile, direct contact).
  • Perform verifications on a clean, preferably offline machine.
  • Keep dated manifest files for audit trail (e.g. manifest-2026-02-28.txt) — avoid publishing internal paths.
